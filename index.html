Process

From Identity to Collective Decision

Five steps to participate in the digital republic. Each step is built on open-source technology, verifiable on-chain, and designed to be censorship-resistant.

1

Generate Your Cryptographic Civic Identity

Use your Iranian National ID Card or Biometric Passport to generate a ZKP key pair via the INID SDK and ZK Passport (Rarimo). Identity proofs are stored on-chain via ERC-7812. Prove you're a unique Iranian citizen without revealing your name, location, or any personal details.

2

Form a Citizen Group (Multi-Sig Trust)

Join or form a small group of 3–7 people who know each other. Open a multi-signature wallet together — funds can only be moved when a threshold of members agree. No single person controls the money.

3

Deliberate on Agora Polis

Join Agora Polis — write your opinion, agree or disagree with others' statements, and an algorithm maps the real clusters of consensus. The result is not a binary — it is a map. Read the Polis whitepaper.

4

Vote Anonymously via Rarimo

Issues from deliberation go to anonymous, verifiable vote via the Freedom Tool on Rarimo through Iranians.Vote. Each voter proves membership without revealing identity. Support for Approval Voting and Ranked Pairs (Tideman).

5

DeFi Treasury Releases Funds

When a proposal reaches ≥5,000 verified citizen votes, the DeXe DAO protocol automatically executes funding. All transactions are on-chain — anyone can see what amount went where and when.

We want to bring these components together, under one app that acts as a citizen-state interface.

ERC-7812

ZK Identity Registry — On-Chain Provable Statements

ERC-7812 defines a singleton on-chain registry for storing provable identity statements. Co-authored by Vitalik Buterin, it uses Sparse Merkle Trees with Poseidon hashing for gas-efficient, cross-chain portable identity proofs.

  • ✓Singleton registry at a deterministic address across all EVM chains
  • ✓Sparse Merkle Tree with Poseidon hash for ZK-friendliness
  • ✓Registrars implement customizable verification logic
  • ✓Cross-chain portability — prove identity anywhere
  • ✓EIP status: Draft — actively evolving standard

Voting

Ranked Pairs — Condorcet-Optimal Elections

Binary yes/no votes lose information. Ranked Pairs (Tideman's method) lets every voter rank all candidates. The algorithm compares every pair head-to-head, sorts by margin of victory, and locks results into a directed graph — skipping any edge that would create a cycle. The source node is the winner.

How It Works

  1. Rank: Each voter ranks all candidates from most to least preferred.
  2. Tally: Every possible pair of candidates is compared head-to-head using all ballots.
  3. Sort: Pairs are ordered by margin of victory — strongest wins first.
  4. Lock: Going down the list, each victory is "locked" into a directed graph — unless it would create a cycle.
  5. Winner: The candidate with no incoming edges (the source) wins.

Implementation: pivot-libre/tideman
